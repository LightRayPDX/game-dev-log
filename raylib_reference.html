GetFrameTime() → float
Returns the time in seconds since the last frame. Multiply velocity by this to make movement frame-rate independent.
Build 01 ▶
Parameters
(none)
Example
float dt = GetFrameTime(); x += speed * dt; // move independent of frame rate
Why this matters: Without delta time, your game runs faster on a faster machine. Multiplying by GetFrameTime() makes sure 200 pixels/second means the same thing on every computer.

BeginDrawing()
Starts the drawing phase for the current frame. Everything drawn goes between this and EndDrawing.
Build 01 ▶
Parameters
(none)
Example
BeginDrawing(); ClearBackground(BLACK); // draw your game here EndDrawing();
↗ official docs
EndDrawing()
Ends the drawing phase and pushes the completed frame to the screen.

Parameters
(none)

↗ official docs
ClearBackground(Color color)
Fills the screen with a solid color. Call first inside BeginDrawing to wipe the previous frame.
Build 01 ▶
Parameters
colorthe background color - use Raylib's named colors: BLACK, WHITE, RAYWHITE, DARKGRAY, etc.
Example
ClearBackground(BLACK);
Why this matters: Without ClearBackground, each frame draws on top of the last one. Your ball leaves a trail instead of appearing to move.

DrawCircle(int centerX, int centerY, float radius, Color color)
Draws a filled circle at a given position.
Build 01 ▶
Parameters
centerXhorizontal center position in pixels - 0 is the left edge of the window
centerYvertical center position in pixels - 0 is the top edge of the window
radiusradius of the circle in pixels
colorfill color - use named colors (RED, BLUE, GREEN) or define a custom Color
Example
DrawCircle(400, 300, 25.0f, RED); // circle at center of 800x600 window DrawCircle((int)x, (int)y, radius, WHITE); // using variables
Note: centerX and centerY are ints but your position variables will likely be floats. Cast them: (int)x.

# game-dev-log

A build-by-build record of learning C++ game development with Raylib.

## Structure

Each build lives in its own folder. The main build document is always `N.0`. Support and reference docs are `N.1`, `N.2`, etc.

```
build-01/    ← game loop, movement, bouncing
build-02/    ← input, collision, state, score
...
```

## Stack

- **Language:** C++
- **Library:** Raylib
- **Platform:** Windows
- **Reference:** learncpp.com

## How it works

Builds get progressively more complex. Each one introduces one new layer on top of what came before. Where it goes from here depends on what's most interesting to build next.
